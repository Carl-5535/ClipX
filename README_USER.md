# ClipX

超轻量级、本地持久化、无条数限制的 Windows 剪贴板增强工具

## 功能特点

- **无限制历史记录** - 不限制剪贴板历史条数，所有数据本地持久化存储
- **开机自启动** - 支持开机自动启动，随时待命
- **标签管理** - 为重要内容添加标签，永久保存，重启不丢失
- **热键唤起** - 按 `F9` 快速调出剪贴板历史浮层
- **搜索功能** - 实时搜索历史剪贴板内容
- **收藏功能** - 标记重要条目为收藏
- **极简设计** - 双进程架构，内存占用 < 10MB
- **隐私安全** - 完全本地运行，无网络请求，数据完全私密

## 使用方法

### 快速开始

1. 解压下载的 ZIP 文件到任意目录
2. 双击运行 `ClipX.exe`
3. 程序会在系统托盘显示图标
4. 开始复制内容，ClipX 自动记录剪贴板历史

### 调出历史记录

- **按 `F9`** - 显示剪贴板历史浮层
- **双击托盘图标** - 显示剪贴板历史浮层

### 浮层操作

| 按键 | 功能 |
|------|------|
| `Enter` | 选中内容，写入剪贴板，关闭浮层 |
| `Esc` | 关闭浮层 |
| `↑` / `↓` | 上下导航选择条目 |
| `Ctrl+F` | 聚焦搜索框 |
| `Delete` | 删除选中条目 |
| `Ctrl+D` | 切换收藏状态 |
| `右键` | 显示上下文菜单（添加标签、删除等）|

### 标签功能

- 给剪贴板条目添加**标签**后，该条目将被**永久保存**
- 未加标签的条目仅保存在**内存**中，重启后自动清除
- 标签是唯一能让内容持久化的方式
- 支持通过标签名称搜索，快速筛选相关条目

### 系统托盘

右键点击系统托盘图标：

- **显示** - 调出剪贴板历史
- **开机自启动** - 切换开机自动启动
- **设置** - 打开设置（开发中）
- **关于** - 查看版本信息
- **退出** - 完全退出程序

## 文件说明

```
ClipX/
├── ClipX.exe        # 主程序（后台守护进程）
├── Overlay.exe      # UI 浮层程序
└── README.md        # 本说明文件
```

首次运行后，程序会在 `%APPDATA%\ClipX\` 目录下创建：

```
%APPDATA%\ClipX/
├── clipx.db         # SQLite 数据库（存储带标签的持久化条目）
├── config.json      # 配置文件
└── logs/
    └── clipx.log    # 日志文件
```

## 数据存储策略

- **内存条目**：普通剪贴板历史仅保存在内存中，重启程序后自动清空
- **持久化条目**：添加标签的条目会保存到数据库，永久保留
- **自动清理**：启动时会自动清理数据库中的孤立标签记录

## 常见问题

**Q: 为什么有些历史记录重启后消失了？**

A: 只有添加了**标签**的条目才会永久保存。未加标签的条目仅保存在内存中，重启后自动清空。这是为了保持数据库精简，只保存真正重要的内容。

**Q: 如何永久保存剪贴板内容？**

A: 在浮层中选中条目后按 `Ctrl+D` 添加标签，或使用右键菜单添加标签。添加标签后内容将永久保存。

**Q: 热键 F9 被其他程序占用了怎么办？**

A: 目前热键固定为 F9。后续版本将支持自定义热键。

**Q: 程序占用多少内存？**

A: ClipX 设计目标内存占用 < 10MB，非常轻量。

**Q: 数据是否会上传到云端？**

A: 完全不会。ClipX 是纯本地工具，所有数据仅存储在你的电脑上，不会进行任何网络请求。
